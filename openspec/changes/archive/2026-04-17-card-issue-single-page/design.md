## Context

- **页面**：`shop-pc/src/views/card/CardIssue.vue`
- **现状**：4步向导，步骤间状态独立，用户无法总览全部信息
- **目标**：保留所有业务逻辑，重构为单页表单 + 实时汇总

## Goals / Non-Goals

**Goals:**
- 单页完成所有发放信息填写
- 右侧汇总卡片实时展示已填内容
- 遵循现有系统 UI 风格（红色 #C00000、Inter 字体）

**Non-Goals:**
- 不改变任何后端 API 行为
- 不修改权限校验逻辑
- 不改变发放业务流程

## Decisions

1. **左侧表单 + 右侧汇总布局**
   - 左侧：会员查询、充值中心、卡号、发放信息（el-input / el-select）
   - 右侧：固定汇总卡片，reactive 显示已填字段
   - 底部：操作栏（重置 + 确认发放）

2. **会员信息只展示手机号 + 等级**
   - 商城 searchMember 接口不返回姓名，移除该字段

3. **面值从卡号详情接口获取**
   - 选择卡号后调用 `getCardDetail` 获取面值，展示在汇总卡片

4. **表单校验在提交时统一检查**
   - 不做步骤级校验，提交时检查所有必填字段

## Risks / Trade-offs

- [Risk] 汇总卡片初始状态为空 → 展示提示"信息不完整"
- [Risk] 面值依赖卡号选择后才有 → 未选卡号时显示"-"
