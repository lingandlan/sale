# 太积堂充值与门店管理系统 - 项目上下文

> 每次新对话开头直接引用此文件，快速了解项目全貌。

## 项目概览

B端管理后台（shop-pc）+ H5端（shop-h5）+ Go后端（backend），用于管理充值中心、门店卡、充值申请审批、用户管理等业务。

## 技术栈

- 后端: Go 1.22 + Gin + GORM + MySQL 5.6 + Redis + Casbin
- 前端 PC: Vue 3 + TypeScript + Element Plus + Pinia + Axios
- 前端 H5: UniApp（早期阶段）
- 使用agent-brower 进行前端验证

## 各环境启动方式

每个环境共用同一个 `start.sh`，端口从 `.env.worktree`（gitignored）读取，**合并时不再冲突**：

| 环境 | 启动命令 | 目录 |
|------|---------|------|
| Main | `./start.sh` | 项目根目录 |
| Alpha | `./start.sh` | `.claude/worktrees/alpha/` |
| Beta | `./start.sh` | `.claude/worktrees/beta/` |

每个目录下维护各自的 `.env.worktree`（不提交到 git）：

```bash
# Main（项目根目录）
FRONTEND_PORT=5175
BACKEND_PORT=8080
REDIS_DB=0

# Alpha（.claude/worktrees/alpha/）
FRONTEND_PORT=5178
BACKEND_PORT=8081
REDIS_DB=1

# Beta（.claude/worktrees/beta/）
FRONTEND_PORT=5179
BACKEND_PORT=8082
REDIS_DB=2
```

脚本原理：
- 读取 `.env.worktree`，不存在则用 Main 默认值（5175/8080/0）
- 前端：写回 `.env.local`，`npx vite` 从中读取端口
- 后端：`APP_SERVER_PORT=XXXX APP_REDIS_DB=X air` 通过环境变量覆盖 config.yaml
- 启动前自动 `lsof -ti:PORT | xargs kill` 清理残留进程

## 关键开发规范

1. GORM model 为唯一 schema 来源，变更必须同步增量 SQL 到 `backend/migrations/sql/`
2. 禁止 `Save()` 做部分更新，必须用 `Updates(map[string]interface{})`
3. Axios interceptor 已 unwrap response.data，代码用 `res.data`
4. Vue SFC 回调中不能用 await，先存变量
5. Element Plus el-dropdown trigger 用数组 `["hover"]`
6. 启动: 各环境用 `./start.sh` 一键启动，端口配置在 `.env.worktree`（gitignored）
7. Vite HMR 需要 `usePolling: true, interval: 1000`（Claude Code 编辑不触发 fsevents）

## 前端页面验证（agent-browser）

验证前端页面前，加载已保存的登录状态，跳过登录流程：

```bash
# 1. 加载登录态（每次验证前执行）
agent-browser state load .claude/browser-auth-main.json

# 2. 直接访问目标页面
agent-browser open http://localhost:5175/<目标页面>
```

如果 token 过期（被跳转到登录页），重新登录并保存：

```bash
agent-browser snapshot -i                    # 填写手机号+密码
agent-browser fill @e6 "13800000000"          # 手机号
agent-browser fill @e7 "123456"               # 密码
agent-browser click @e4                       # 登录
agent-browser wait --load networkidle
agent-browser state save .claude/browser-auth-main.json
```
